Game map / level editor

The same kernel as the cadastre example — with no basemap, no @fleximap/plugin-topology (so no JSTS in the bundle), and a custom world-units CRS registered at runtime through crs.register(). Nothing in @fleximap/core was changed to make a tilemap possible. That is the point of the entire library.

Palette

Click the map to place. Everything lands on the grid.

Mode

Box-select: drag a marquee. Shift adds to the selection. Delete removes it.

Procedural generation

A commit-middleware hook. Place a building: trees and rocks appear around it, in the same command — so one undo removes all of it.

History

Ctrl/Cmd + Z works on the canvas too.

What to notice

        
    
          
  • 
            Snapping is middleware, not a tool feature. The place tool reads
            ctx.lngLat; the snap plugin rewrote it to a tile centre before any tool saw
            the event. The place tool has never heard of the grid — which is why a tool written by a
            stranger next year snaps for free.
          
    • 
          
  • 
            tile-grid is a layer type this preset registered. The kernel ships
            vector and raster and knows nothing about tiles. A deck.gl
            layer would be added exactly the same way.
          
    • 
          
  • 
            Generation is the same seam as cadastral validation. Both are commit middleware.
            The kernel does not know that one scatters crates and the other prevents a lawsuit.
          
    • 
          
  • 
            The coordinates are world units, not degrees. A custom CRS makes every planar
            facility in the kernel — snap, quantise, distance, area — speak tiles, with no change to
            any of them.
