Urban planning / zoning

urbanPlanningPreset() — the same kernel and the same plugins as the cadastre example, with different numbers and one inverted severity.

Topology issues here are warnings, not errors. Draw a zone that overlaps another one: it is allowed, stored, and merely flagged. This is the same topology plugin, and the same topology.overlap rule, that the cadastre example uses to refuse the write — with a different severity, decided by the preset, not by the plugin. The plugin finds the overlap; only the domain knows whether an overlap is a dispute or a thought. live: topology.overlap severity = …

What to try

  1. Draw a new zone — click corners, double-click to close.
  2. Pick a category in the attribute form below. The polygon recolours instantly: one layer, one match expression, N categories.
  3. Draw a second zone that overlaps an existing one. Watch it be stored anyway and flagged in Topology.
  4. Snapshot the plan as scenario A, change some zoning, snapshot as B, then compare them.

Vertices snap to the 5 m planning grid and to existing corners. The draw tool has never heard of the snap plugin — snapping is interaction middleware that rewrote the pointer position before any tool saw it. Hold Alt to suppress it.
